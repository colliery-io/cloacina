## Key technical decisions (durable)
- **Reclaim by re-addressing, not cancel+reschedule** — keeps `task_execution_id` so the FleetExecutor's `oneshot` rendezvous (FleetCoordinator) receives the new agent's result with zero changes.
- **OQ-2 (exactly-once) stance:** match the thread executor's at-least-once posture on heartbeat-loss (deliberate per initiative). 3×-missed-heartbeat (45s) is the liveness guard; a partitioned-but-alive agent may double-execute — same as threads.
- **Tenant = the task's namespace tenant**, not the agent's. Selection parses namespace first; "public" namespace ↔ `None` agent tenant.
- **Saturation needs no code** — composition handles it.
