## Objective

`GET /ready` currently only checks Postgres connection health. It should also check `ReactiveScheduler::list_graphs()` — if any graph has `running: false` (reactor JoinHandle finished unexpectedly), `/ready` should return 503. This integrates with k8s liveness probes so crashed reactors trigger pod restarts.

### Key files
- `crates/cloacinactl/src/commands/serve.rs` — `ready()` handler, add `reactive_scheduler.list_graphs()` check
