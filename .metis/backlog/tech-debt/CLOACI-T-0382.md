## Objective

Connection-level reactor authZ is wired (403 on upgrade if key not authorized). But per-command checks are not — once connected, any command is accepted. Need to pass `auth.key_id` into `process_reactor_command` and call `registry.check_reactor_op_auth()` before dispatching each command. The method exists and is unit-tested, just not called from the handler.

### Key files
- `crates/cloacinactl/src/server/ws.rs` — `handle_reactor_socket` and `process_reactor_command` need `key_id` param
- `crates/cloacina/src/computation_graph/registry.rs` — `check_reactor_op_auth` already implemented
