## Objective

`ReactiveScheduler` stores `JoinHandle`s and can detect finished tasks via `is_finished()`, but has no background loop that automatically restarts them. Need a supervision task that `select!`s on all handles, detects unexpected completion (panic or error), re-spawns the task using the stored `ComputationGraphDeclaration`, and re-registers the new channel sender in the `EndpointRegistry`.

### Key files
- `crates/cloacina/src/computation_graph/scheduler.rs` — add `supervise()` method or background task in `load_graph`
- `AccumulatorFactory::spawn` already supports re-creation from the same factory
