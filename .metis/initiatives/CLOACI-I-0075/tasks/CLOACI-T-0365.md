# Python accumulator decorators (@stream_accumulator, @passthrough_accumulator)

## Objective

Implement Python decorators for accumulators that mirror the Rust accumulator macros. Each decorator wraps a Python function into the Rust `Accumulator` trait implementation with `spawn_blocking` for GIL safety.

Note: This task depends on the Rust accumulator trait existing (I-0074). If I-0074 hasn't started yet, this task should define the Python-side decorator API and test it with a mock Rust backend, then wire to the real trait when I-0074 lands.

## Acceptance Criteria

## Acceptance Criteria

## Acceptance Criteria

- [ ] `@cloaca.passthrough_accumulator` decorator — wraps a Python function that transforms an event into a boundary. Socket-only, no event loop.
- [ ] `@cloaca.stream_accumulator(type="kafka", topic="...")` decorator — wraps a Python function as a stream-backed accumulator. Event loop generated by Rust side.
- [ ] `@cloaca.stream_accumulator(type="kafka", topic="...", state=dict)` — stateful variant with checkpointing
- [ ] All Python accumulator `process()` calls wrapped in `spawn_blocking` (GIL)
- [ ] Decorators register the accumulator with a global registry (same pattern as `@task`)
- [ ] Python function signature: `def name(event) -> boundary` for stateless, `def name(event, state) -> (boundary, state)` for stateful
- [ ] Decorated functions produce types compatible with `BoundarySender` serialization

## Implementation Notes

Follow the pattern in `crates/cloacina/src/python/task.rs` — `@task` uses `#[pyfunction]` to register a decorator that creates a `PyTaskWrapper`. The accumulator decorators similarly create a `PythonAccumulatorWrapper` that implements the Rust `Accumulator` trait (or a compatible interface if the trait doesn't exist yet from I-0074).

Place in `crates/cloacina/src/python/accumulator.rs` (new file).

T-0364 (graph executor — establishes the Python↔Rust bridge pattern for computation graphs). Rust Accumulator trait from I-0074 (can be mocked if not available).
